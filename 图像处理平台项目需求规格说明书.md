<h2 id="1-引言">1. 引言</h2>

<div class="md-section-divider"></div>

<h3 id="11-目的">1.1 目的</h3>

<p>本文编写的目的是引导开发人员进行概要设计和详细设计。</p>

<p>本文首先从总体结构上给出整个系统的轮廓，然后又对功能需求、性能需求和其它非功能性需求进行了详细的描述。这些文字和图形都为了本文档能详细准确地描述用户的需求，同时也为用户更容易地理解这些需求的描述创造了条件。</p>

<p>本文详尽说明了这一软件产品的需求和规格，这些规格说明是进行设计的基础，也是编写测试用例和进行系统测试的主要依据。同时，也是用户确定软件功能需求的主要依据。</p>

<div class="md-section-divider"></div>

<h3 id="12-文档格式">1.2 文档格式</h3>

<p>本着简单易学、跨平台的原则，便于后续开发阅读、分享、协作、差异比较和版本控制，本文以 Markdown 格式编写，发布在项目工程所在的 Wiki 中。</p>

<div class="md-section-divider"></div>

<h3 id="13-预期的读者和阅读建议">1.3 预期的读者和阅读建议</h3>

<p>本文档的主要内容共分4部分：系统概述、功能性需求、非功能性需求和外部接口描述。系统概述部分主要对系统的整体结构进行了大致的介绍；功能性需求部分对系统的功能需求进行了详细描述，是本文的主要部分；非功能性需求部分对性能、安全性、可用性方面进行了详细的描述；外部接口需求部分对用户界面、软件接口、硬件接口和通讯接口等进行了描述。</p>

<p>本文档面向多种读者对象：</p>

<ul>
<li>项目经理：项目经理可以根据该文档了解预期产品的功能，并据此进行系统设计、项目管理。</li>
<li>架构师：对需求进行分析，并设计出系统，包括数据库的设计。</li>
<li>程序员：配合《设计文档》，了解系统功能，编写《用户手册》。</li>
<li>测试员：根据本文档编写测试用例，并对软件产品进行功能性测试和非功能性测试。</li>
<li>销售人员：了解预期产品的功能和性能。</li>
<li>用户：了解预期产品的功能和性能，并与分析人员一起对整个需求进行讨论和协商。</li>
<li>其他人员：如部门领导、公司领导等可以据此了解产品的功能和性能。</li>
</ul>

<p>在阅读本文档时，首先要了解产品的功能概貌，然后可以根据自身的需要对每一功能进行适当的了解。</p>

<div class="md-section-divider"></div>

<h3 id="14-范围">1.4 范围</h3>

<li>西南实训中心与理想具有相对稳定的项目团队，不稳定的团队将影响项目的进度和质量。</li>
</ul>

<h2 id="3-系统功能性需求">3. 系统功能性需求</h2>

<div class="md-section-divider"></div>

<h3 id="31-系统角色">3.1 系统角色</h3>

<p>三类主要用户：教师、学员、网站管理人员，主要功能如下：</p>

<p>教师：</p>

<ul>
<li>创建课程：支持 Unity 3D 格式类型的课程，内容可以上传；  </li>
<li>课程设置：进行课程的目标、适合人群等设置；</li>
<li>学员管理：可以监控学员学习情况，通过公告、问答、讨论区以及私信与学员进行沟通。</li>
</ul>

<p>学员：</p>

<ul>
<li>注册网站账号：以多种方式注册登录网站；社会学员在报名审核通过后，和校内学员有同样的权限；未注册用户只能浏览网站的公开信息；  </li>
<li>学习工具辅助：通过笔记、问答、考试等巩固课程学习成果；  </li>
<li>课程评价：学员可以对课程进行评价，方便新注册学员决定是否加入报名；  </li>
<li>个人主页及课程中心：记录课程历程及各种学习信息。</li>
</ul>

<p>网站管理人员：</p>

<ul>
<li>课程管理：可以对课程进行全方位管理，可以进行推荐、审核等，可以管理课程评价、讨论、笔记、问答等各种辅助功能；  </li>
<li>用户管理：系统可以设置四种权限的用户，包括超级管理员、管理员、教师和学员，可添加新用户，可手动增加或移除课程的学员；  </li>
<li>内容管理：可以对网站的页面、可编辑部分进行编辑和管理；  </li>
<li>系统设置：超级管理员可以进行各项站点设置、升级、备份操作、封禁IP等。</li>
</ul>

<h3 id="61-课程">6.1 课程</h3>

<p>使用者：管理员、系统管理员。 <br>
目的：课程维护。 <br>
功能概述：通过在线学习的方式，把学校相关的专业课程开放出来，供学员使用、交流。学员可通过一系列的工具进行课程的学习和与网站其他用户的交互。此功能在业务上和实训中的“我的教学”职能相同，但又多出一些课程全局性的设置，这部分是不对老师开放的。</p>

<div class="md-section-divider"></div>

<h4 id="611-课程管理">6.1.1 课程管理</h4>

<p>对课程进行新增，查询，管理，推荐，预览，发布和关闭，删除操作。</p>

<p><img src="http://git.oschina.net/uploads/images/2014/0820/151547_d8051d07_109968.png" alt="6.1.1" title=""></p>

<ul>
<li>新增：点击右上角【创建课程】，创建课程，如何创建课程请参阅“5.6 我的教学”。</li>
<li>查询：输入查询条件（课程分类，课程状态，标题，创建者）点击【搜索】可以查到相应的课程条目。</li>
<li>管理：点击后会进入单个课程的编辑页面。详细的课程编辑功能请查看“5.6 我的教学”。</li>
<li>推荐：点击此按钮会使该课程处在课程列表中的【推荐】类目中第一名的位置。用于课程运营之用（和首页模板相关）。</li>
<li>预览：点击相应功能，可作为未报名/已报名用户查看课程页面，检查课程对不同人群的呈现效果。</li>
<li>发布和关闭课程：点击发布课程，课程即可在前台显示，注册用户即可浏览、和学习该课程；当课程状态为已发布后，该功能变为关闭课程功能，点击后即可关闭课程，注册用户，需要注意，关闭后已加入该课程的学员仍然可查看该课程内容，而未加入该课程的注册用户点击该课程会显示课程已关闭，无法进行学习等操作。</li>
<li>删除：可删除相应的课程条目，课程的下属内容也会一并被删除，包括课程内容，资料，和与之相匹配的讨论帖子。</li>

<h3 id="71-站点设置">7.1 站点设置</h3>

<p><img src="http://git.oschina.net/uploads/images/2014/0820/172853_6c9fccae_109968.png" alt="7.1" title=""></p>

<ul>
<li>可填写网站的基本信息，包括网站名称和网站副标题、网站域名、网站LOGO、浏览器图标、SEO关键词和SEO描述信息、管理员邮箱地址、ICP备案号、首页模版、网站统计分析代码部署。</li>
<li>网站名称和网站副标题：填写相应文字，当用户访问网站时，网页标题中会显示所填内容，网站导航左边会显示所填内容。</li>
<li>网站域名：网站域名是您网站的地址，用户通过这个地址来访问您的网站，一般不需要改动。</li>
<li>网站LOGO：LOGO会在网站导航左边的点击【上传】，选择相应的图片。提示：LOGO图片的高度建议不要超过50px。当上传网站LOGO后，导航左侧的网站名称便会换成网站LOGO，【上传】按钮旁边会出现【删除】按钮，点击【删除】即可删除网站LOGO。</li>
<li>浏览器图标：操作方式同“网站LOGO”，设置成功后，访问网站任何页面，都可以在浏览器中显示网页标题的地方看到这个LOGO图片。在多标签浏览器中，通常是在标签页标签的头部。提示：需要上传.ico格式的图标文件，而不是一般的图片文件。</li>
<li>SEO关键词和SEO描述信息：将需要优化的关键词组成比较通顺的语句，填写到SEO描述信息中。</li>
<li>管理员邮箱地址：填写网站管理员的邮箱，此邮箱将在用户访问网站出错的时候显示出来，让用户有机会联系站长。</li>
<li>ICP备案号：填写该网站域名的备案号，此备案号会显示在网站页面底端。</li>
<li>网站统计分析代码部署：可添加网站统计分析代码，监测网站数据。</li>
</ul>

<blockquote class="white-blockquote">
  <p>统计代码是什么：统计代码是网站统计软件发布的一段代码，用以统计添加统计代码的网站的数据，比较常用的网站统计软件有谷歌分析、百度统计、腾讯分析、CNZZ等。请自行查看相关软件的使用方法，当您使用任意一个软件时，必须将其提供的统计代码添加到此栏内，您才能正常使用其提供的统计功能。 <br>
  统计代码有什么用：使用统计软件可以让您了解在网站上的用户行为的具体数据，如每天有多少访客访问网站，有多少网站会员登录网站，新旧用户比例，有多少新用户来到网站，都是从哪里来的，有多少浏览量，网站停留时间等，通过这些数据您可以知道您的网站的整体状况，也可以知道网站哪些内容受欢迎，根据各种用户的数据，让您能够更加了解用户。也可以从侧面了解如运营效果，推广效果。可参阅的网站分析工具文章，百度官方出品：《网站分析白皮书》了解更多。</p>
</blockquote>

<h3 id="72-全局设置">7.2 全局设置</h3>

<div class="md-section-divider"></div>

<h4 id="721-课程设置">7.2.1 课程设置</h4>

<p>对课程的全局开关进行控制。可以设置是否给新学员发送欢迎私信以及私信内容，可以控制是否在购买时需要填写提交个人资料。</p>

<p><img src="http://git.oschina.net/uploads/images/2014/0820/173337_b4aeca19_109968.png" alt="7.2.1" title=""></p>

<ul>

<li>加入课程时填写个人资料：开启后，学员加入课程时，需填写个人资料。</li>
</ul>

<div class="md-section-divider"></div>

<h4 id="722-邮件服务器设置">7.2.2 邮件服务器设置</h4>

<p>对发送邮件的 SMTP 服务器进行配置。只有设置成功此项，网站的邮件发送功能才会被启用可在各大企业邮箱提供商处免费注册使用，注册及使用方法因提供商而稍有不同，可自行查看。</p>

<p><img src="http://git.oschina.net/uploads/images/2014/0820/173414_84989247_109968.png" alt="7.2.2" title=""></p>

<h3 id="73-用户相关设置">7.3 用户相关设置</h3>

<div class="md-section-divider"></div>

<h4 id="731-注册设置">7.3.1 注册设置</h4>

<p><img src="http://git.oschina.net/uploads/images/2014/0820/174045_f5c0fbac_109968.png" alt="7.3.1" title=""></p>

<ul>
<li>开放注册设置：设置是否允许用户在网站上注册账号。</li>
<li>新用户激活邮件设置：即用户注册后，通过邮件通知用户注册成功。可以在邮件内容中填写一些介绍性或者指引性的文字。</li>
<li>欢迎信息设置：此处填写的信息会以站内私信，而不是邮件的方式发送给新注册用户。</li>
</ul>

<div class="md-section-divider"></div>

<h4 id="732-登录设置">7.3.2 登录设置</h4>

<p><img src="http://git.oschina.net/uploads/images/2014/0820/174052_80ad83d8_109968.png" alt="7.3.2" title=""></p>

<li>用户登录限制：此选项开启后，同一帐号只能在一处登录，以防止账号被公用。</li>
<li>第三方登录：只有在开启状态下，才能支持第三方登录。目前我们支持微博账号、QQ账号、人人网账号三种方式。</li>
</ul>

<p>若发现恶意访问者，可以在这里添加他登录的IP，提交后通过此IP将不能访问您的网站。一行一个IP，被加入黑名单的IP将被禁止访问！暂不支持网段的封锁。</p>

<div class="md-section-divider"></div>

<p>可查询所有重要操作的操作记录。 <br>
查询：输入查询条件（等级、起止时间、模块名、操作名、用户名箱），点击【搜索】按钮可以查询到相应结果。显示结果包括：用户名、操作名、日志等级、操作时间、IP地址。</p>

<h2 id="8-非功能性需求">8. 非功能性需求</h2>

<div class="md-section-divider"></div>

<h3 id="81-性能需求">8.1 性能需求</h3>

<ul>
<li>客户端一般响应时间（除报表统计、数据导入）不超过 3 秒。</li>
<li>报表统计时间不超过 30 秒。</li>
<li>支持 2000 名学生信息的一次性导入，导入时间不超过 300 秒。</li>
<li>支持 500 名用户并发使用门户系统，并保证性能不受影响。</li>
<li>支持 100 名用户并发使用直播、点播系统，并保证性能不受影响。</li>
</ul>

<div class="md-section-divider"></div>

<h3 id="82-安全性需求">8.2 安全性需求</h3>

<ul>
<li>权限控制：根据不同用户角色，设置相应权限，用户的重要操作都做相应的日志记录以备查看，没有权限的用户禁止使用系统。学生只可查看自已的学习成绩，教师只可查看所教课程的学生成绩</li>
<li>重要数据加密：本系统对一些重要的数据按一定的算法进行加密，如用户口令、重要参数等。</li>
<li>数据备份：允许用户进行数据的备份和恢复，以弥补数据的破坏和丢失。</li>
<li>记录日志：本系统应该能够记录系统运行时所发生的软件错误。这些错误记录便于查找错误的原因。日志同时记录用户的关键性操作信息。</li>
</ul>

<div class="md-section-divider"></div>

<h3 id="83-可用性需求">8.3 可用性需求</h3>

<ul>
<li>控制必录入项：本系统能够对必须录入的项目进行控制，使用户能够确保信息录入的完整。同时对必录入项进行有效的统一的提示。</li>
<li>容错能力：系统具有一定的容错和抗干扰能力，在非硬件故障或非通讯故障时，系统能够保证正常运行，并有足够的提示信息帮助用户有效正确地完成任务。</li>
<li>操作完成时有统一规范的提示信息：例如删除操作时，系统可提示警示框“您确认删除记录吗？操作不可恢复！”，用户点击确认后，系统才执行删除操作，删除后可直接返回相关页面。</li>
</ul>

<div class="md-section-divider"></div>

<h3 id="84-用户文档">8.4 用户文档</h3>

<p>同本软件一起发行的用户文档包括：</p>

<ul>
<li>安装手册：pdf 格式文件。</li>
<li>用户手册：pdf 格式文件。</li>
</ul>

<h3 id="85-其它需求">8.5 其它需求</h3>

<li>系统安装方便，易于维护。</li>
</ul>
